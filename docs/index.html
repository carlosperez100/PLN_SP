Qué se corrió, en una frase

Se comparó, sobre 14,853 epicrisis reales de MIMIC-IV (12,785 pacientes), tres enfoques de PLN para clasificar la naturaleza del evento adverso (8 clases del Anexo 02 GG-ESSALUD-2021): un modelo léxico clásico (TF-IDF con regresión logística y con LinearSVC) frente a un transformer clínico (Bio_ClinicalBERT). El pipeline está escrito en Python con scikit-learn, gensim/NLTK y Hugging Face Transformers.

Todo el código es público. Los 7 scripts y el informe de resultados están versionados en GitHub y se pueden abrir y leer sin credenciales. Lo único que no se publica es el texto clínico (dataset .parquet) y el modelo entrenado (.pkl), porque el Data Use Agreement (DUA) de MIMIC-IV lo prohíbe — eso es una exigencia ética/legal, no una omisión.

Los scripts de Python (en orden de ejecución)

3
fase3_corpus_expansion.py Corpus

Construye el corpus. Recorre las 331,793 notas de alta de MIMIC-IV y detecta eventos con Tier A (223 códigos ICD-10), Tier B (35 patrones de texto / expresiones regulares) y filtro NegEx para descartar negaciones ("sin infección"). Salida: 14,863 notas candidatas + muestra estratificada de 350 notas para anotación experta.

4
fase4_entrenar_baseline.py OE2 · Modelos

El corazón del trabajo. Une el texto de discharge.csv.gz con la etiqueta débil, vectoriza con TF-IDF (palabra 1–2 gramas, 30,000 features, class_weight="balanced") y entrena/compara Regresión Logística vs LinearSVC. Guarda métricas y el mejor modelo.

4
fase4_clinicalbert.py OE2 · BERT

El tercer modelo: Bio_ClinicalBERT (emilyalsentzer, Hugging Face) como extractor de embeddings congelado (mean-pooling) + cabezal LogReg, sobre submuestra estratificada. Demuestra la factibilidad del transformer sin GPU.

4
fase4_crossval.py OE2 · Robustez

Validación cruzada 5-fold (StratifiedKFold) que reporta media ± desviación de accuracy, F1-ponderado, F1-macro y Kappa. Prueba la config reforzada TF-IDF palabra + char n-gramas (3–5).

4
fase4_split_paciente.py OE2 · Rigor

Corrección metodológica: rehace todo con split a nivel de paciente (GroupShuffleSplit por subject_id) para evitar fuga de datos, y añade CV agrupada (StratifiedGroupKFold) + matriz de confusión.

4
fase4_circularidad.py OE2 · Auditoría

Experimento de honestidad científica: enmascara (__MASK__) los 35 disparadores Tier B y reentrena, para medir cuánto del acierto viene del patrón literal vs. del contexto. Resultado: solo cae 1.35 pts.

4
fase4_multietiqueta.py OE2 · Multi-label

Reformula la tarea como multi-etiqueta (una nota puede tener varias naturalezas, el 30.9% las tiene) con TF-IDF + OneVsRestClassifier(LinearSVC). Métricas: F1-micro/macro, Hamming loss, exact-match.

Resultados obtenidos

Comparación de modelos (validación cruzada 5-fold, con partición por paciente). Fuente: RESULTADOS_FASE4.md.

ModeloExactitudF1-macroKappa
TF-IDF + Regresión Logística (baseline)0.6280.4660.474
★ TF-IDF + LinearSVC (palabra + char)0.7310.5150.581
Bio_ClinicalBERT congelado + LogReg (sin fine-tuning)0.380.190.18
Hallazgo principal: en una tarea definida por patrones léxicos, el modelo clásico superó al transformer clínico congelado. La hipótesis inicial (ClinicalBERT > baseline) se refutó: sin fine-tuning, BERT quedó último. La lectura honesta es el F1-macro (~0.51), no el F1-ponderado, y la etiqueta débil tiene circularidad estructural → las cifras miden consistencia con la regla, no validez clínica. Reformulación multi-etiqueta: F1-micro 0.751.

Cómo se corrió (entorno y comandos)

Todo se ejecutó localmente en Windows con Anaconda (Python 3.13). Estas son las librerías y los comandos exactos:

# Entorno: Anaconda 3 (Python 3.13) — 100% software libre
# Librerías: scikit-learn 1.7.2 · torch 2.12.1 (CPU) · transformers 5.13.0 · pandas · nltk · gensim

# Variable necesaria en Windows para evitar el conflicto de OpenMP:
set KMP_DUPLICATE_LIB_OK=TRUE

# 1) Construir el corpus desde MIMIC-IV (Tier A/B + NegEx)
python fase3_corpus_expansion.py

# 2) Entrenar y comparar los modelos (OE2)
python fase4_entrenar_baseline.py     # TF-IDF + LogReg vs LinearSVC
python fase4_crossval.py              # validación cruzada 5-fold
python fase4_clinicalbert.py          # Bio_ClinicalBERT (descarga ~440 MB la 1.ª vez)
python fase4_split_paciente.py        # split por paciente (sin fuga)
python fase4_circularidad.py          # experimento de enmascaramiento
python fase4_multietiqueta.py         # reformulación multi-etiqueta

Opción A — Leer/mostrar el código (inmediata)

Todos los scripts se abren directamente en GitHub (enlaces de la sección anterior). Se puede hacer scroll por fase4_entrenar_baseline.py y ver el TF-IDF, el LinearSVC y el cálculo de métricas línea por línea. No requiere nada instalado.

Opción B — Re-ejecutar el pipeline

Requiere acceso credencializado a MIMIC-IV (ver abajo). Con la carpeta codigo/ clonada y el discharge.csv.gz en su ruta, se corre con los comandos de arriba. El dataset y el modelo no viajan en el repo por el DUA — se regeneran localmente.

Nota de reproducibilidad honesta: cualquiera puede leer el 100% del código, pero para volver a correrlo se necesita la credencial de PhysioNet (curso CITI + firma del Data Use Agreement). Es la misma barrera ética que protege los datos de pacientes; se explica en la ficha técnica de la presentación.
